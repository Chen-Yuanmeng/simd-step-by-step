# SIMD step-by-step

这是中国科学院大学 (UCAS) 并行程序设计课程的作业.

## 任务描述

通过不同的方式实现 SIMD 优化, 比较不同方式实现的时间和性能差异, 并作简要并行方法和性能分析.

## SIMD 实现方式

1. 编译器自动向量化
2. 通过在源码中加入 language extensions 指导编译器进行向量化
3. 直接使用 SIMD intrinsics 实现
4. 手写 SIMD 汇编代码实现

## 性能分析

| 编号 | 具体优化方式 | Windows 加速比 | Linux 加速比 |
| --- | --- | --- | --- |
| 0-00 | 标量，无任何优化 | 1 | 1 |
| 1-01 | 仅开通用优化 | 10.84 | 10.67 |
| 1-02 | 启用AVX2指令集 | 26.83 | 19.27 |
| 1-03 | 启用AVX512F指令集 | 26.22 | 16.28 |
| 2-01 | 语言扩展指导向量化 | 26.33 | 28.26 |
| 3-01 | SSE4.1 intrinsics (128-bit) | 18.72 | 23.76 |
| 3-02 | AVX2 intrinsics (256-bit) | 25.55 | 33.48 |
| 3-03 | AVX512F intrinsics (512-bit) | 29.3 | 30.58 |
| 4-01 | GCC 内联汇编 SSE (128-bit) | 11.68 | 12.75 |
| 4-02 | GCC 内联汇编 AVX2 (256-bit) | 25.78 | 26.84 |
| 4-03 | GCC 内联汇编 AVX512F (512-bit) | 25.21 | 27.65 |

## 结论

- SIMD 优化方法相较于标量版本, 在性能上有显著提升, 加速比可达数十倍.
- 不同的 SIMD 优化方法在不同平台上的性能表现存在差异, 这可能与编译器的优化策略和虚拟机的调度有关.
- 在大部分平台和场景下, 使用编译器自动向量化的结果已经足够; 如果需要精确控制 SIMD 指令的使用, 可以使用对应的 SIMD 库 (如 Intel Intrinsics, 或者 ARM Neon 等) 进行优化. 手写内联汇编虽然可以实现 SIMD优化, 但由于其固有的性能代价和维护难度, 通常不建议使用.
